The pitch

Software has always had a brutal prioritization filter.

For the 10 features the engineering team can build this sprint, there are 300 they can't. Small tools that would save you 20 minutes a day. The dashboard view only two people need. The filter you've been asking for since last year.

It's not that engineering doesn't want to build those things — the math just never works.

We're entering the era of personal software. Tools that could never justify their own roadmap slot, but exist anyway because one person needed them. You're not replacing engineering. You're filling in the long tail of things their prioritization process was never going to reach.

Cowork was about orchestrating the tools you already have. Claude Code is about building the tools that don't exist because nobody was ever going to build them for you.

Getting into Claude Code

If you've never opened it before, here's the quickest path:

  • Open the Claude Desktop App.
  • In the left sidebar, click Code.
  • Pick or create a folder on your computer for this project (e.g. ~/Desktop/week6-tool). That folder is your workspace — everything Claude builds lives there.
  • You should see a chat input at the bottom. That's where you prompt. You're in.

The best way to learn is to dive in and self-teach. Watch a few YouTube walkthroughs to get the shape of it, and keep a second Claude chat window open alongside your build — one you can ask questions to whenever you get stuck or want to understand what's going on.

If anything's broken at this step, DM Danny before you rage-quit.

The exercise

Build something you'd actually use every week.

Not a demo. Not a toy. Something that earns a spot in your real workflow — the kind of thing you'd open Monday morning out of habit.

Scope it to roughly 2 hours of focused build time. If you're going past that, you're overbuilding. It runs locally on your machine — no need to deploy anything this week.

If you're staring at a blank page, these lanes are good starting points. You don't have to pick one.

  • Resurrect a Cowork dead-end. Something you tried to automate but couldn't quite get there.
  • A feature you wish existed in a tool you use every day. Fill the gap without replacing the tool.
  • A recurring Claude chat, turned into a real tool. If you paste the same thing every week, it's a UI waiting to happen.
  • A tool your team would use. Built for you, useful for them.
Required workflows — do these every time
  • 1
    Start in plan mode.
    Hit Shift+Tab until you see "plan mode." Claude proposes what it's about to do before touching a file. Read the plan. If it's wrong, push back now — not after it's written 400 lines in the wrong direction.
  • 2
    Write a short CLAUDE.md before your first prompt.
    Three sections: Who this is for, What it does, The stack (e.g. "Next.js, Tailwind, no database, run locally"). This anchors every conversation. If you don't lock the stack, Claude will improvise differently every session.
  • 3
    Run it after every meaningful change.
    Don't stack five prompts and then test. Every change → refresh the browser → does it still work? Problems are 10x easier to fix when they're the only thing that just changed.
  • 4
    Ask Claude to review its own work before you call it done.
    "Before we ship this, review what you built. What could break? What's fragile? What would you clean up?" You'll get a free round of polish without having to think of every issue yourself.
Helpful tips
  • 1
    Show, don't tell, for UI.
    If you want a specific look, screenshot something you like — a page, a component, a vibe — and drop the image in. Works 10x better than describing it.
  • 2
    Screenshots of errors work great.
    If something's broken in the browser, screenshot the page or the console and paste it in. Claude can usually diagnose from the image.
  • 3
    If you've been stuck for 20 minutes, stop and reset.
    Tell Claude: "This approach isn't working. Let's throw it out and try something simpler." Then describe the simpler version. Digging deeper on a broken approach is the #1 time-killer.

Why this matters
